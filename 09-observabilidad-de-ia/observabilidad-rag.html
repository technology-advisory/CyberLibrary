Observabilidad RAG

Mide retrieval, fuentes, calidad del contexto y groundedness. Esta guía combina una explicación entendible para perfiles no técnicos con controles y evidencias útiles para arquitectura, seguridad, cumplimiento y auditoría.

En pocas palabras

Observar RAG requiere medir recuperación y generación por separado. Hay que saber qué consulta se ejecutó, qué filtros se aplicaron, qué documentos llegaron al contexto y si la respuesta quedó respaldada por ellos.

Cómo abordarlo paso a paso

Capturar la consulta transformada

Registra reescrituras y expansión semántica.

Medir retrieval

Observa top-k, puntuaciones, filtros, diversidad y fuentes.

Evaluar groundedness

Comprueba si afirmaciones relevantes se apoyan en contexto.

Controlar frescura

Relaciona documentos con versión y fecha de indexación.

Detectar fallos de autorización

Alerta sobre recuperaciones bloqueadas o cross-tenant.

Checklist práctica

  • Existe evidencia de: trazas de retrieval.
  • Existe evidencia de: dataset de preguntas.
  • Existe evidencia de: resultados de groundedness.
  • Existe evidencia de: alertas de acl.

Evidencias que conviene conservar

  • Trazas de retrieval
  • Dataset de preguntas
  • Resultados de groundedness
  • Alertas de ACL

Conserva únicamente la evidencia necesaria, con acceso restringido, versión y fecha. La trazabilidad no debe crear un nuevo repositorio de datos sensibles.

Errores que suelen aparecer

  • Medir solo la respuesta final
  • Registrar texto completo de documentos sensibles
  • No distinguir fallo de recuperación de fallo de generación

Indicadores útiles para este tema

  • Recall@k
  • Groundedness
  • Edad media del índice
  • Recuperaciones bloqueadas por ACL
