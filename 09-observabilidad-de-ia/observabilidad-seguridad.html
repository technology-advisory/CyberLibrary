09.04 · Observabilidad de IA

Observabilidad de seguridad

Detecta inyección, jailbreak, abuso, exfiltración y comportamientos anómalos. Esta guía combina una explicación entendible para perfiles no técnicos con controles y evidencias útiles para arquitectura, seguridad, cumplimiento y auditoría.

En pocas palabras

La observabilidad de seguridad para IA combina señales de identidad, contenido, red, herramientas y comportamiento. Debe detectar secuencias, no solo palabras aisladas en prompts.

Cómo abordarlo paso a paso

Definir casos de uso

Prioriza inyección, fuga, abuso de tools, extracción y uso no autorizado.

Normalizar eventos

Unifica gateway, IAM, RAG, MCP, agentes y DLP.

Crear correlaciones

Combina varias señales débiles dentro de una sesión.

Añadir contexto de riesgo

Enriquece con criticidad, usuario, datos y permisos.

Probar detecciones

Usa simulaciones controladas y mide cobertura y ruido.

Checklist práctica

  • Existe evidencia de: catálogo de casos de uso.
  • Existe evidencia de: mapeo de fuentes.
  • Existe evidencia de: reglas siem.
  • Existe evidencia de: resultados purple team.

Evidencias que conviene conservar

  • Catálogo de casos de uso
  • Mapeo de fuentes
  • Reglas SIEM
  • Resultados purple team

Conserva únicamente la evidencia necesaria, con acceso restringido, versión y fecha. La trazabilidad no debe crear un nuevo repositorio de datos sensibles.

Errores que suelen aparecer

  • Buscar solo términos prohibidos
  • No incluir tool calls en la correlación
  • Automatizar bloqueo sin contexto de negocio

Indicadores útiles para este tema

  • Cobertura de técnicas
  • Falsos positivos por regla
  • Tiempo de detección
  • Casos probados con éxito
