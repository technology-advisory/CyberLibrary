07.04 · Arquitectura segura

Patrones API

Centraliza acceso, políticas, cuotas, observabilidad y protección de modelos. Esta guía combina una explicación entendible para perfiles no técnicos con controles y evidencias útiles para arquitectura, seguridad, cumplimiento y auditoría.

En pocas palabras

Las APIs de IA deben tratar prompts y respuestas como cargas sensibles y no confiables. Un AI Gateway centraliza identidad, cuotas, políticas, observabilidad, routing y protección de datos entre aplicaciones y proveedores.

Cómo abordarlo paso a paso

Centralizar entrada

Evita integraciones directas dispersas con proveedores.

Autenticar aplicación y usuario

Propaga identidad para aplicar políticas y atribuir consumo.

Aplicar políticas de datos

Clasifica, redacta, bloquea o tokeniza antes de enviar.

Validar respuesta

Controla esquema, tamaño, contenido activo y acciones derivadas.

Diseñar resiliencia

Usa timeouts, circuit breakers, fallback y límites de coste.

Controles específicos

ControlObjetivo concretoEvidencia
Gateway obligatorioImpedir bypass de controles comunes.Rutas y reglas de red.
Schema de salidaAceptar solo estructuras esperadas.Validación JSON/contrato.
Cuota contextualLimitar por usuario, aplicación y modelo.Políticas de rate limit.
Routing aprobadoEnviar datos solo a regiones y modelos permitidos.Reglas de selección.

Checklist práctica

  • Existe evidencia de: catálogo de apis.
  • Existe evidencia de: políticas de gateway.
  • Existe evidencia de: esquemas de respuesta.
  • Existe evidencia de: pruebas de fallback.

Evidencias que conviene conservar

  • Catálogo de APIs
  • Políticas de gateway
  • Esquemas de respuesta
  • Pruebas de fallback

Conserva únicamente la evidencia necesaria, con acceso restringido, versión y fecha. La trazabilidad no debe crear un nuevo repositorio de datos sensibles.

Errores que suelen aparecer

  • Guardar claves del proveedor en aplicaciones
  • Aplicar DLP después de enviar
  • Reintentar indefinidamente ante fallo

Indicadores útiles para este tema

  • Tráfico que pasa por gateway
  • Respuestas rechazadas por esquema
  • Coste por aplicación
  • Circuit breakers activados
