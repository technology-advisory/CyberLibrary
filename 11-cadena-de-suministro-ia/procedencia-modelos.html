11.02 · Cadena de suministro de IA

Procedencia de modelos

Verifica origen, versiones, licencias, firmas y cambios del modelo. Esta guía combina una explicación entendible para perfiles no técnicos con controles y evidencias útiles para arquitectura, seguridad, cumplimiento y auditoría.

En pocas palabras

La procedencia de modelos documenta origen, creador, licencia, versiones, transformaciones, validación y cadena de custodia. Un nombre de repositorio no demuestra que el artefacto desplegado sea auténtico.

Cómo abordarlo paso a paso

Verificar fuente

Descarga solo desde repositorios o proveedores aprobados.

Capturar identidad

Registra digest, firma, versión, arquitectura y licencia.

Documentar transformaciones

Anota cuantización, fine-tuning, merging y conversión de formato.

Validar antes de promover

Ejecuta seguridad, calidad y compatibilidad en entorno aislado.

Mantener revocación

Permite bloquear versiones comprometidas en todos los entornos.

Checklist práctica

  • Existe evidencia de: ficha de procedencia.
  • Existe evidencia de: hashes y firmas.
  • Existe evidencia de: licencia y condiciones.
  • Existe evidencia de: resultados de validación.

Evidencias que conviene conservar

  • Ficha de procedencia
  • Hashes y firmas
  • Licencia y condiciones
  • Resultados de validación

Conserva únicamente la evidencia necesaria, con acceso restringido, versión y fecha. La trazabilidad no debe crear un nuevo repositorio de datos sensibles.

Errores que suelen aparecer

  • Confiar en el nombre del archivo
  • Perder linaje tras cuantización
  • No conservar la versión original

Indicadores útiles para este tema

  • Modelos con firma
  • Derivados con padre identificado
  • Intentos de desplegar revocados
  • Tiempo de validación
