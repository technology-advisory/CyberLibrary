11.03 · Cadena de suministro de IA

Procedencia de datasets

Documenta fuentes, permisos, calidad, transformaciones y restricciones de uso. Esta guía combina una explicación entendible para perfiles no técnicos con controles y evidencias útiles para arquitectura, seguridad, cumplimiento y auditoría.

En pocas palabras

La procedencia de datasets debe explicar de dónde vienen los datos, con qué permisos se obtuvieron, cómo se transformaron y qué versiones se usaron. Sin linaje no puede reproducirse ni impugnarse una evaluación.

Cómo abordarlo paso a paso

Registrar fuentes y derechos

Documenta origen, licencia, consentimiento y restricciones.

Versionar snapshots

Congela la colección exacta utilizada en entrenamiento o prueba.

Trazar transformaciones

Anota limpieza, deduplicación, etiquetado, filtrado y generación sintética.

Evaluar calidad y representatividad

Mide cobertura, errores, duplicados y grupos infrarepresentados.

Gestionar retirada

Permite localizar y eliminar datos cuya autorización cambia.

Checklist práctica

  • Existe evidencia de: datasheet.
  • Existe evidencia de: snapshot versionado.
  • Existe evidencia de: código de transformación.
  • Existe evidencia de: registro de licencias.

Evidencias que conviene conservar

  • Datasheet
  • Snapshot versionado
  • Código de transformación
  • Registro de licencias

Conserva únicamente la evidencia necesaria, con acceso restringido, versión y fecha. La trazabilidad no debe crear un nuevo repositorio de datos sensibles.

Errores que suelen aparecer

  • Descargar datos públicos y asumir uso libre
  • Modificar datos sin versionar
  • No separar entrenamiento y evaluación

Indicadores útiles para este tema

  • Datasets con licencia clara
  • Transformaciones reproducibles
  • Duplicados detectados
  • Solicitudes de retirada completadas
