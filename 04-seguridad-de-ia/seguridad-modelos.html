04.05 · Seguridad de IA

Seguridad de modelos

Protege pesos, endpoints, datos de entrenamiento y disponibilidad. Esta guía combina una explicación entendible para perfiles no técnicos con controles y evidencias útiles para arquitectura, seguridad, cumplimiento y auditoría.

En pocas palabras

Proteger modelos implica controlar acceso a pesos, endpoints, gradientes, datasets, artefactos y capacidad de consulta. Las amenazas incluyen extracción, inversión, poisoning, evasión y denegación de servicio.

Cómo abordarlo paso a paso

Clasificar el activo

Determina sensibilidad de pesos, arquitectura, datos y propiedad intelectual.

Limitar consultas

Aplica autenticación, cuotas y detección de patrones de extracción.

Proteger el pipeline

Verifica datos, código, dependencias y artefactos antes de entrenar o desplegar.

Evaluar robustez

Prueba entradas adversariales y degradación fuera de distribución.

Preparar respuesta

Define rotación, retirada, reentrenamiento o restricción del endpoint.

Controles específicos

Control de acceso a pesosEvitar copia no autorizada.Permisos y logs de repositorio.
Detección de extracciónIdentificar consultas sistemáticas.Alertas por patrón y volumen.

Checklist práctica

  • Existe evidencia de: clasificación del modelo.
  • Existe evidencia de: logs de acceso a artefactos.
  • Existe evidencia de: resultados adversariales.
  • Existe evidencia de: plan de respuesta a extracción.

Evidencias que conviene conservar

  • Clasificación del modelo
  • Logs de acceso a artefactos
  • Resultados adversariales
  • Plan de respuesta a extracción

Conserva únicamente la evidencia necesaria, con acceso restringido, versión y fecha. La trazabilidad no debe crear un nuevo repositorio de datos sensibles.

Errores que suelen aparecer

  • Exponer probabilidades detalladas sin necesidad
  • Entrenar con fuentes no verificadas
  • No distinguir error normal de ataque sistemático

Indicadores útiles para este tema

  • Consultas anómalas
  • Artefactos con firma válida
  • Tasa de éxito adversarial
  • Tiempo de contención del endpoint
