05.02 · Auditoría de IA

Auditoría técnica de IA

Examina arquitectura, configuración, flujos, permisos, registros y comportamiento. Esta guía combina una explicación entendible para perfiles no técnicos con controles y evidencias útiles para arquitectura, seguridad, cumplimiento y auditoría.

En pocas palabras

La auditoría técnica de IA examina arquitectura, configuración, código, datos, integraciones y comportamiento. Debe combinar revisión documental, pruebas controladas y observación de telemetría.

Cómo abordarlo paso a paso

Definir reglas de compromiso

Acordar alcance, entornos, datos permitidos, límites y parada de emergencia.

Revisar arquitectura y amenazas

Relaciona componentes con trust boundaries y controles.

Inspeccionar configuración

Evalúa identidades, redes, secretos, filtros, logging y dependencias.

Ejecutar pruebas reproducibles

Registra payload, versión, contexto, resultado y evidencia.

Priorizar por explotabilidad e impacto

Distingue debilidad teórica de ruta de ataque viable.

Checklist práctica

  • Existe evidencia de: reglas de compromiso.
  • Existe evidencia de: diagramas revisados.
  • Existe evidencia de: evidencias de pruebas.
  • Existe evidencia de: matriz de hallazgos y retest.

Evidencias que conviene conservar

  • Reglas de compromiso
  • Diagramas revisados
  • Evidencias de pruebas
  • Matriz de hallazgos y retest

Conserva únicamente la evidencia necesaria, con acceso restringido, versión y fecha. La trazabilidad no debe crear un nuevo repositorio de datos sensibles.

Errores que suelen aparecer

  • Probar sin versión exacta del sistema
  • Confundir comportamiento inesperado con vulnerabilidad
  • Publicar secretos en capturas o logs

Indicadores útiles para este tema

  • Hallazgos reproducibles
  • Cobertura de componentes
  • Tiempo de remediación
  • Hallazgos reabiertos
