10.01 · Seguridad Cloud para IA

Azure OpenAI

Construye un baseline seguro para Azure OpenAI y servicios asociados. Esta guía combina una explicación entendible para perfiles no técnicos con controles y evidencias útiles para arquitectura, seguridad, cumplimiento y auditoría.

En pocas palabras

Una arquitectura segura de Azure OpenAI combina identidad administrada, redes privadas, control de datos, políticas de despliegue, logging y límites de consumo. Las claves estáticas deben ser la excepción.

Cómo abordarlo paso a paso

Diseñar landing zone

Separa suscripciones, redes, identidades, logs y responsabilidades.

Usar autenticación Entra ID

Prefiere managed identities y RBAC sobre claves compartidas.

Privatizar conectividad

Aplica Private Endpoint, DNS y bloqueo de acceso público cuando proceda.

Centralizar consumo

Pasa aplicaciones por API Management o gateway con cuotas y políticas.

Monitorizar configuración y uso

Envía diagnósticos, métricas y cambios a Log Analytics/SIEM.

Checklist práctica

  • Existe evidencia de: diagrama azure.
  • Existe evidencia de: roles y managed identities.
  • Existe evidencia de: políticas de red.
  • Existe evidencia de: configuración de diagnósticos.

Evidencias que conviene conservar

  • Diagrama Azure
  • Roles y managed identities
  • Políticas de red
  • Configuración de diagnósticos

Conserva únicamente la evidencia necesaria, con acceso restringido, versión y fecha. La trazabilidad no debe crear un nuevo repositorio de datos sensibles.

Errores que suelen aparecer

  • Compartir API keys entre aplicaciones
  • Crear Private Endpoint sin resolver DNS correctamente
  • Permitir despliegues fuera de regiones aprobadas

Indicadores útiles para este tema

  • Llamadas con Entra ID
  • Recursos con acceso público
  • Cambios de deployment
  • Consumo por aplicación
