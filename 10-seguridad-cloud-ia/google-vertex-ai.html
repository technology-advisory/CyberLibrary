10.03 · Seguridad Cloud para IA

Google Vertex AI

Asegura Vertex AI con perímetros, IAM, cifrado, logging y gobierno de datos. Esta guía combina una explicación entendible para perfiles no técnicos con controles y evidencias útiles para arquitectura, seguridad, cumplimiento y auditoría.

En pocas palabras

Vertex AI requiere un diseño coherente de proyectos, service accounts, VPC Service Controls, CMEK, endpoints y logging. Los notebooks y pipelines suelen ser tan sensibles como los propios modelos.

Cómo abordarlo paso a paso

Estructurar proyectos

Separa datos, desarrollo, entrenamiento y serving según confianza.

Usar service accounts dedicadas

Evita cuentas por defecto y claves descargables.

Crear perímetros VPC-SC

Reduce exfiltración entre proyectos y servicios.

Controlar notebooks y pipelines

Restringe imágenes, redes, secretos y ejecución.

Activar auditoría y postura

Revisa Cloud Audit Logs, Security Command Center y cambios IAM.

Checklist práctica

  • Existe evidencia de: diagrama de proyectos.
  • Existe evidencia de: iam por service account.
  • Existe evidencia de: perímetros vpc-sc.
  • Existe evidencia de: logs de auditoría.

Evidencias que conviene conservar

  • Diagrama de proyectos
  • IAM por service account
  • Perímetros VPC-SC
  • Logs de auditoría

Conserva únicamente la evidencia necesaria, con acceso restringido, versión y fecha. La trazabilidad no debe crear un nuevo repositorio de datos sensibles.

Errores que suelen aparecer

  • Usar el proyecto por defecto para todo
  • Crear claves JSON de larga duración
  • Dejar notebooks con salida abierta a Internet

Indicadores útiles para este tema

  • Claves de service account activas
  • Violaciones VPC-SC
  • Endpoints privados
  • Cambios IAM de alto riesgo
