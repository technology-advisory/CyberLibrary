08.02 · AI Red Teaming

Jailbreaks

Evalúa la resistencia de políticas, guardrails y moderación sin confundir seguridad con censura. Esta guía combina una explicación entendible para perfiles no técnicos con controles y evidencias útiles para arquitectura, seguridad, cumplimiento y auditoría.

En pocas palabras

Las pruebas de jailbreak evalúan la resistencia de políticas de uso y guardrails frente a evasión, role-play, fragmentación, transformación y presión conversacional. Deben distinguir seguridad del contenido y seguridad del sistema.

Cómo abordarlo paso a paso

Definir políticas evaluadas

Asocia cada prueba a una conducta concreta que se pretende impedir.

Crear variaciones semánticas

Cambia idioma, formato, rol, pasos y contexto sin repetir una frase.

Medir consistencia

Repite con temperatura, longitud y sesiones distintas.

Evaluar sobrebloqueo

Comprueba que solicitudes legítimas no se rechazan innecesariamente.

Ajustar por capas

Combina políticas, clasificadores, diseño de producto y revisión humana.

Controles específicos

ControlObjetivo concretoEvidencia
Dataset equilibradoIncluir casos prohibidos y permitidos cercanos.Corpus etiquetado.
Pruebas multilingüesDetectar brechas por idioma.Resultados por lengua.
Consistencia estocásticaRepetir suficientes veces.Distribución de resultados.
Medida de utilidadControlar falsos positivos.Tasa de rechazo legítimo.

Checklist práctica

  • Existe evidencia de: catálogo de políticas.
  • Existe evidencia de: corpus de jailbreaks.
  • Existe evidencia de: resultados repetidos.
  • Existe evidencia de: análisis de falsos positivos.

Evidencias que conviene conservar

  • Catálogo de políticas
  • Corpus de jailbreaks
  • Resultados repetidos
  • Análisis de falsos positivos

Conserva únicamente la evidencia necesaria, con acceso restringido, versión y fecha. La trazabilidad no debe crear un nuevo repositorio de datos sensibles.

Errores que suelen aparecer

  • Publicar payloads sensibles sin control
  • Optimizar solo para un benchmark
  • Confundir negativa verbal con ausencia de riesgo

Indicadores útiles para este tema

  • Attack success rate
  • Falsos positivos
  • Variación entre idiomas
  • Regresiones tras actualización
