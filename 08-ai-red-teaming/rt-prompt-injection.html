08.01 · AI Red Teaming

Prompt Injection

Diseña pruebas seguras para inyección directa, indirecta y persistente. Esta guía combina una explicación entendible para perfiles no técnicos con controles y evidencias útiles para arquitectura, seguridad, cumplimiento y auditoría.

En pocas palabras

El red teaming de prompt injection comprueba si instrucciones directas o indirectas pueden alterar objetivos, revelar contexto o activar herramientas. La prueba debe medir impacto, no solo conseguir que el modelo diga una frase prohibida.

Cómo abordarlo paso a paso

Definir activos objetivo

Selecciona system prompt, datos RAG, herramientas, secretos y decisiones.

Construir familias de payloads

Incluye conflictos, encoding, instrucciones en documentos y cadenas multilingües.

Variar contexto y rol

Prueba usuarios, sesiones, fuentes y permisos diferentes.

Observar efectos

Registra no solo la respuesta, sino retrieval, tool calls y datos enviados.

Convertir hallazgos en regresión

Añade casos reproducibles al pipeline de evaluación.

Checklist práctica

  • Existe evidencia de: payloads versionados.
  • Existe evidencia de: configuración exacta.
  • Existe evidencia de: trazas completas.
  • Existe evidencia de: casos de regresión.

Evidencias que conviene conservar

  • Payloads versionados
  • Configuración exacta
  • Trazas completas
  • Casos de regresión

Conserva únicamente la evidencia necesaria, con acceso restringido, versión y fecha. La trazabilidad no debe crear un nuevo repositorio de datos sensibles.

Errores que suelen aparecer

  • Contar “jailbreaks” sin impacto
  • Probar solo en chat vacío
  • No repetir tras cambios de modelo

Indicadores útiles para este tema

  • Tasa de éxito por familia
  • Inyecciones con tool call
  • Fugas de contexto
  • Casos corregidos que no regresan
