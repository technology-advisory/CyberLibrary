08.05 · AI Red Teaming

MCP Supply Chain

Evalúa riesgos de servidores, cambios maliciosos y confianza en herramientas. Esta guía combina una explicación entendible para perfiles no técnicos con controles y evidencias útiles para arquitectura, seguridad, cumplimiento y auditoría.

En pocas palabras

El red teaming de la cadena MCP simula servidores o paquetes que cambian después de ser confiados: nuevas tools, descripciones engañosas, scopes ampliados, tool shadowing y exfiltración mediante parámetros aparentemente legítimos.

Cómo abordarlo paso a paso

Establecer versión confiable

Captura manifest, tool schemas, hash, dependencias y permisos iniciales.

Introducir cambios controlados

Añade parámetros, cambia descripciones o sustituye una dependencia.

Comprobar detección

Valida si CI/CD, registro o cliente identifica la diferencia.

Intentar abuso silencioso

Prueba exfiltración, acceso lateral y llamadas no visibles al usuario.

Evaluar revocación

Mide cuánto tarda en bloquearse el servidor y rotarse sus secretos.

Controles específicos

ControlObjetivo concretoEvidencia

Firma de paqueteImpedir sustitución del artefacto.Verificación de firma.
Scope driftAlertar cuando aumentan permisos.Historial de scopes.
Canary de exfiltraciónDetectar salida a destinos no aprobados.Alerta de red controlada.

Checklist práctica

  • Existe evidencia de: baseline del servidor.
  • Existe evidencia de: artefacto modificado.
  • Existe evidencia de: alertas generadas.
  • Existe evidencia de: timeline de revocación.

Evidencias que conviene conservar

  • Baseline del servidor
  • Artefacto modificado
  • Alertas generadas
  • Timeline de revocación

Conserva únicamente la evidencia necesaria, con acceso restringido, versión y fecha. La trazabilidad no debe crear un nuevo repositorio de datos sensibles.

Errores que suelen aparecer

  • Probar solo el código fuente inicial
  • No considerar actualizaciones del registry
  • Compartir credenciales con el entorno de prueba

Indicadores útiles para este tema

  • Cambios detectados antes de ejecución
  • Tiempo hasta revocación
  • Exfiltraciones bloqueadas
  • Scopes ampliados identificados
